# Release v0.9.2

Fixes `.ntd` and `.nts` parsing for bare string values that start with text
that looks like a primitive value.

- Fixed UUID-like bare strings such as
  `9f006820-df12-4456-8ce4-1bc96a2a3fcc` being partially parsed as the number
  `9`, which caused compilation to fail when more content followed.
- Required number, boolean, and null values to end at a real value boundary
  before they are parsed as primitives.
- Kept bare strings such as `false-positive` and `nullish` as strings instead
  of treating their prefixes as booleans or null.
- Applied the same primitive-boundary handling to request body and header values
  in `.nts` files.
- Added compiler coverage for primitive-looking bare string prefixes in
  definition files.
- Simplified terminal input value and cursor-position resolution across query,
  view, search, edit, and AI modes.
