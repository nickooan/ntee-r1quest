# Release v0.21.1

A polish release: the terminal app gets a real color identity. The whole UI
moves from the terminal's stock ANSI palette to a gruvbox-dark theme — the same
look as ntee-editor — and the response pane stops rendering replies as a wall
of monochrome text: labels, status codes, headers, and JSON bodies are now
syntax-highlighted, in live responses and history alike. Resuming an AI session
no longer buries you under the replayed transcript, suggestion lists become
fully walkable instead of stopping at the first screenful, and the bundled
Claude Code plugin's skills get a fact-check-and-tighten pass against the
actual runtime.

- **Gruvbox-dark theme across the app.** Every style — panes, borders, file
  tree, selections, search matches, suggestion popups, edit-mode status, AI
  modal, session picker, the permission banner — now draws from one truecolor
  gruvbox-dark palette (cream foreground, warm grays, the classic
  red/green/yellow/aqua accents) instead of ad-hoc 16-color ANSI codes.
  Matching ntee-editor's style, the two apps finally look like siblings;
  termenv degrades gracefully to 256 colors on lesser terminals. The cursor
  and selected rows switch from terminal-dependent reverse video to explicit
  foreground-on-background pairs, so selection looks the same everywhere.
- **The response pane gets syntax highlighting.** Formatted responses and
  history entries are colored by line shape: section rules dimmed, `URL` /
  `Method` / `Status` / `Headers` / `Body` labels in bold aqua, the status code
  green for 2xx, yellow for 3xx, red for 4xx/5xx, header keys in aqua — and
  pretty-printed JSON bodies with green keys, gray punctuation, and cream
  values. Applied identically to the live response pane and the history view;
  the r1quest DSL highlighter is untouched and deliberately not applied to
  response output.
- **Suggestion lists are fully walkable.** The file-suggestion popup was capped
  at 8 entries total, so ↑/↓ ran out of list even when a typed directory had
  far more matches. The cap is now 200 — the popup still renders a small
  window around the selection, but arrow keys can walk through every match.
  Cached typed-history suggestions keep the old popup-sized limit of 8.
- **Resumed AI sessions start with a clear pane.** Resuming a session used to
  bottom-pin the entire replayed transcript, leaving the impression of a
  conversation already in full swing. Now the replayed history renders only
  its tail in the top ~30% of the pane, ending at the history divider, with
  the rest left clear for the new conversation. The anchor releases the moment
  you scroll, send a prompt, or a live turn streams in — returning to the
  normal bottom-pinned transcript.
- **Claude Code plugin skills v0.3.0.** The seven bundled skills were audited
  against the actual CLI source and tightened so an agent can follow them
  deterministically. Corrections: the previously undocumented
  `session-cleanup-period` config key, the `patch` method missing from the
  `type` list, a multipart example that contradicted the content-type rule,
  and references to a formatter/compile-only command that doesn't exist.
  Stricter rules: `--init` is flagged as interactive (agents write the home
  config directly instead), GraphQL selection depth and pagination arguments
  are now hard rules, `.ntd` key placement has a crisp shared-vs-specific
  threshold, joint-chain env mapping requires exact key matches, temp
  `.r1q-task-*` joint files must always be deleted, and mutations are never
  executed without explicit confirmation.
