# Release v0.21.0

Finding things gets fast, and talking to the agent gets smarter. Query mode
learns real fuzzy search — type any part of a request name and nested `.nts`
files pop up from collapsed folders, full path shown. The AI chat learns file
references: a standalone `#keyword` fuzzy-searches the whole request tree,
promotes to a compact `[name]` pill, and travels to the agent as a proper ACP
`resource_link` instead of a raw path pasted into prose. Typing while the
agent is thinking now does the right thing per agent — injected into the live
turn on Claude, queued and delivered next turn on Codex/Cursor — and the
thinking indicator finally tells the truth, staying on for exactly as long as
the turn actually runs. Permission requests stop hiding in the status line.

- **Fuzzy search in query mode.** Suggestions are no longer prefix-only: the
  popup now ranks exact > prefix > substring > subsequence, so `orders` finds
  `folder-2/sub-folder-1/get-orders-by-id` from anywhere — and so does `gob`
  (skipped-letter matching against name and path). The search corpus is a full
  recursive walk of the request root, so requests inside collapsed directories
  are found without expanding anything; suggestions always display the full
  relative path, and Enter executes the pick directly. Query-mode suggestions
  stay scoped to `.nts` requests. Recently-called entries keep their green
  cache styling even when the file suggestion absorbs the duplicate history
  row, and cached-input dedup is normalized (case and `\`/`/` variants never
  show twice).
- **`#file` references in AI chat.** Typing a standalone `#keyword` in `@ai`
  opens a fuzzy popup over the whole tree — directories, `.nts`, and `.ntd`
  files alike. Enter/Tab promotes the match to a `[filename]` or `[dir-name]`
  pill (colliding names fall back to the relative path); Esc declines and
  leaves the text literal. The trigger is deliberately strict: `# x` and
  `abc#x` never fire, and the popup only shows while the cursor is inside the
  token. On send the pill stays readable in the transcript while each
  referenced file rides along as an ACP `resource_link` content block
  (`file://` URI + name) — a first-class file reference the agent understands,
  instead of an absolute path inlined into the message (which some agents
  mis-parse; a leading path read as a slash command in Codex is what prompted
  the change). Wired through all three adapters (Claude, Codex, Cursor).
- **A prompt translation pipeline.** Enter-send now runs the typed input
  through ordered stages that produce what the transcript shows and what the
  agent receives: `/name args` custom-command expansion first (unchanged
  behavior, relocated), then reference-pill attachment — so a pill passed as a
  `/cmd [file]` argument still resolves. New macro kinds become new stages.
- **Type while the agent thinks — steering or queue, per adapter.** Enter
  mid-turn no longer races the runtime. Adapters declare whether they accept
  prompts during a live turn: on Claude the message is sent immediately and
  injected into the running turn (true steering — the agent incorporates it
  mid-run); on Codex/Cursor, where a concurrent prompt would interrupt the
  running turn, tips queue locally as faint `⏳ queued:` rows above the input
  and deliver as one merged follow-up turn the moment the current turn
  completes — entering the transcript at actual send time. Queued tips keep
  their `[file]` pills, survive leaving AI mode (the session keeps running),
  and are dropped with a notice on error, stop, or `@reload`. `@`-commands
  still act immediately.
- **The thinking indicator lasts the whole turn.** Previously it switched off
  after a 3-second quiet window between streamed chunks — so a long-running
  agent looked "done" while still working — and a 30-second cap could kill
  long turns outright. The indicator is now tied to the actual ACP turn
  lifecycle: on from prompt-send until the turn completes (the `ai/prompt`
  request resolving with its stop reason), errors, or the session stops. No
  timers, no guessing, no cap on agentic runs — and short turns clear
  instantly. A per-turn id keeps a superseded turn's late completion from
  clearing a newer one (which is exactly what Claude's steering hand-off
  emits).
- **Permission requests you can't miss.** The tool-approval prompt grows from
  one plain status row into a three-row banner: a black-on-yellow
  `⚠ PERMISSION REQUEST` badge, the requested action in bold, and color-coded
  actions — `[y] allow` in green, `[n] reject` in red.
- **Fix: the response pane scrolls all the way down.** An off-by-one between
  the layout and the scroll clamp (query mode's hint row wasn't subtracted)
  left the last line of a response — typically the closing `}` — permanently
  below the fold. The clamp now mirrors the rendered pane exactly.
- **Fix: one-shot output ends with a newline.** `r1q -p <request>` no longer
  glues the shell prompt onto the closing `}` of the response body. CLI-only:
  joint-chain output, the error path, and the content handed to an open app
  were already correct and are unchanged.
