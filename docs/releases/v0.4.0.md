# Release v0.4.0

Adds local AI agent chat through ACP adapters, configurable AI provider
selection, and a cleaner terminal/file-management structure.

- Added `@ai` mode for short in-place AI chats while working in
  `ntee-r1quest`, without replacing your regular local AI agent terminal app.
- Added Codex ACP and Claude Code ACP adapters with matching session, prompt,
  permission, error, and lifecycle behavior.
- Added `-ai codex` and `-ai claude` CLI support, plus `.r1qconfig.json` `ai`
  config support. CLI `-ai` takes precedence over config, and `codex` remains
  the default.
- Added unsupported-adapter validation so unknown AI adapter names fail with a
  clear error.
- Kept AI sessions alive when pressing Esc, so Esc only hides the AI overlay and
  the chat/session can be resumed later.
- Added `@exit` and `@quit` app exit commands, including cleanup that stops the
  active AI adapter before the app exits.
- Added AI chat history persistence across overlay close/reopen, with the chat
  reopening at the bottom.
- Added an AI pending animation while waiting for an AI response.
- Added cursor editing in AI chat input, including left/right cursor movement,
  insertion at the cursor, and backspace before the cursor.
- Improved AI permission request modal rendering with a solid background.
- Added shared runtime app name/version constants and a `version:bump` script
  that updates `package.json`, `package-lock.json`, and runtime version data.
- Added Claude Code and Codex ACP integration tests.
- Refactored file tree, path, command, formatting, and view-file helpers into
  `src/runtime/file-manager`.
- Split terminal UI helpers and components out of `terminal-app.tsx` into
  focused files under `src/views/terminal`.
- Updated README documentation for AI mode, adapter selection, config fallback,
  and the intended quick-action role of the embedded AI chat.
