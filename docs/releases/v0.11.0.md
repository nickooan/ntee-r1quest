# Release v0.11.0

Adds configurable custom AI commands, makes search mode quicker to enter and
leave, and keeps the terminal app smooth during typing and AI streaming.

- Added `custom-ai-commands` config support: define reusable AI prompts with a
  `name`, optional `description`, and `instruction`, then invoke them in AI mode
  as `/name arg1 arg2` with positional `$1`, `$2`, ... substitution.
- Showed a command suggestion popup while typing `/` in AI mode, with arrow-key
  navigation and Tab or Enter to accept the highlighted command.
- Scanned custom AI commands on boot and re-scanned config from disk on
  `@reload` instead of reusing the cached boot snapshot.
- Allowed entering search with an inline query, such as `@s uuid` or
  `@search order id`, to highlight matches immediately.
- Remembered the previous mode when entering search so Esc returns to it, and
  removed search from the Shift+Tab quick-switch cycle.
- Showed a "Nothing found" overlay when a search query has no matches, dismissed
  with Enter.
- Kept typing and scrolling smooth by no longer re-rendering response content,
  search highlights, and the file sidebar on every keystroke or cursor blink.
- Avoided re-reading the collection from disk while typing a request path by
  caching directory listings until they change.
- Kept long AI responses responsive by handling streamed updates in linear
  rather than quadratic time.
- Documented custom AI commands, inline search, and config source precedence and
  merge rules in the README.
- Added test coverage for custom commands, AI command input, file-tree caching,
  and config parsing.
