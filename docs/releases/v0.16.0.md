# Release v0.16.0

Replaces LMDB with **ntee-db** — a purpose-built, pure-Go embedded database —
as the storage engine behind all local history and caching, and rebuilds the
API-call history on top of it: per-call records, secondary indexes, richer
queries, and automatic retention. The database ships inside the npm package as
a native library with prebuilds for macOS and Linux.

- **New embedded database — ntee-db.** A small, stdlib-only, log-structured
  key-value store living in the repo as its own Go module: an
  append-only JSONL main table as the source of truth, an in-memory primary-key
  index with prefix scans, a blob side file for large values, crash-safe
  recovery (torn-tail truncation, tombstone replay), an index-hint snapshot for
  fast boots, and compaction. Fully unit-tested.
- **LMDB removed.** The runtime cache (input suggestions, API call history, AI
  sessions) now runs on ntee-db through a new Node binding. Existing caches
  start fresh — the cache is best-effort, so nothing migrates and nothing
  breaks.
- **`@ntee/ntee-db` Node binding, bundled in the package.** The Go core is
  compiled to a C-shared library and loaded in-process via koffi. It ships
  inside the npm tarball (bundled dependency, no extra install) with prebuilt
  natives for macOS arm64 and Linux amd64/arm64. Platforms without a prebuild
  degrade gracefully: the cache is disabled, the app is fully functional.
- **Secondary indexes with rich queries.** Records can be indexed by named
  string/number attributes (explicit values or derived from the record). Exact
  lookups take a `±N` limit (first/last N), value-**prefix** search applies the
  limit per distinct value — "the latest call of every matching endpoint" is
  one bounded query — and numeric ranges are supported.
- **API history re-architected.** Every call is its own time-keyed record
  (nothing is overwritten), indexed by `endpoint` and `traceId` — powering
  latest-per-endpoint history, in-order trace filtering (`@h <traceId>`), and
  endpoint-prefix search.
- **Automatic retention.** An index can cap records per value (`maxPerValue`):
  the endpoint index keeps the **5 most recent calls per endpoint**, evicting
  older ones automatically as durable deletes. Range delete by primary key
  (`RemoveByPkLess` / `RemoveByPkGreater`) enables time-based pruning; primary
  keys are designed so lexical order = time order.
- **Build & packaging.** `npm run build` (and `prepack`) now also compiles the
  database's native library; a Docker-based script cross-builds the Linux
  natives — including x86-64 — from Apple Silicon. All shipped binaries are
  stripped.
- Fixed the custom-command UI displaying incorrectly.
