# Release v0.12.0

Adds a persistent on-disk cache for request history and typed inputs, a new
History mode to browse past API calls, and input suggestions while typing in
query and view modes.

- Added an embedded LMDB cache at `~/.ntee-r1quest/cache` that stores recent
  request/response history and typed query/view inputs; it runs in-process with
  no external service, and writes are best-effort so a cache failure never
  breaks the app.
- Recorded each successful request — request and response details plus
  duration — keyed by endpoint and HTTP method, so the same path called with
  different methods (`/orders [get]` vs `/orders [post]`) is tracked separately
  and a repeat call overwrites the previous entry.
- Labelled GraphQL requests by operation, such as `CreatePost [mutation]`
  instead of `/api [post]`, derived from the operation name and type in the
  request body.
- Cached query and view inputs on Enter (excluding `@` commands) and showed a
  suggestion overlay above the command line while typing, mixing
  current-directory files and folders (yellow) with matching cached inputs
  (green), prefix-matched.
- Added History mode via `@history` or `@h`: the left pane lists cached
  endpoints under "Endpoints", the right pane shows a clear, sectioned
  request/response view, and the header reports the call's duration.
- Filtered endpoints by typing in History mode with a suggestion overlay; Up and
  Down navigate it and Enter selects, Shift+Up/Down move the endpoint highlight,
  and Up/Down scroll the Results pane when no filter is active.
- Allowed entering search from History, such as `@s query`, to highlight matches
  within the Results, returning to History on Esc.
- Added `@clean-cache` and its `@cc` alias to erase the cache, with a
  confirmation overlay dismissed by Enter.
- Persisted cache writes before one-shot CLI runs (`-p`) exit, so requests run
  this way — and the requests behind external events — also appear in history.
- Added test coverage for endpoint labelling (including GraphQL), input and
  endpoint suggestions, the History results formatter, and the new mode and
  cache commands.
