# Release v0.16.2

Migrates the Codex AI adapter off its deprecated upstream package. No behavior
change for users — same adapter, same configuration — but installs are clean
again and the Codex integration is back on a maintained release line.

- **Codex ACP package migrated.** `@zed-industries/codex-acp` (0.14) was
  deprecated upstream in favor of `@agentclientprotocol/codex-acp`; the
  adapter now depends on the new package (1.1). The replacement is pure
  JavaScript — the per-platform binary sub-packages
  (`@zed-industries/codex-acp-darwin-arm64`, …) are gone, and `npm install`
  no longer prints deprecation warnings.
- **Wire compatibility verified.** The new server speaks the same ACP
  handshake (`protocolVersion: 1`, ndjson over stdio), so `-ai codex` works
  unchanged. Upstream 1.x also advertises capabilities the adapter can grow
  into later (session resume/list, MCP servers over HTTP, image prompts).
