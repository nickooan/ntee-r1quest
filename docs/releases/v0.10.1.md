# Release v0.10.1

Adds Cursor CLI as an AI adapter and improves AI mode feedback when an ACP
agent cannot start or exits unexpectedly.

- Added a Cursor ACP adapter that starts the installed Cursor CLI with
  `agent acp`.
- Registered `cursor` as a supported `-ai` adapter alongside `codex` and
  `claude`.
- Added Cursor as an option in the interactive config generator and allowed
  home config YAML to store `ai: cursor`.
- Added a `start:cursor` development script for running the app with the Cursor
  adapter.
- Kept AI mode open and displayed `AI is offline` when an ACP child process
  fails during startup or exits unexpectedly.
- Updated AI plugin documentation with Cursor CLI skill installation commands
  for global and project-local skills.
- Added integration and unit coverage for Cursor ACP startup, adapter
  selection, config generation, CLI adapter resolution, and the AI offline
  message.
