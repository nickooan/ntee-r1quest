# Release v0.18.0

Scales the embedded history store (`ntee-db`) past the small-cache regime and
moves its unbounded reads off the event loop. History behaves the same for users
— the cache just holds its speed as it grows and no longer blocks the UI thread
on a large record fetch. The embedded binding moves to `@ntee/ntee-db` 0.3.0
(the record/batch reads are now async — the one breaking change).

- **Record and batch reads run off the event loop.** `getMany`,
  `secIndexRecords`, `secIndexPrefixRecords`, and `prefixScanRecords` are now
  `Promise`-returning — their result sets are unbounded, so the native read
  happens on a worker thread instead of stalling the JS loop. Point-scale calls
  stay synchronous where a Promise would cost more than the work (`get`, `has`,
  `secIndexHas`, `stats`, key-only index queries, `put`). The app's history
  layer awaits these (`listApiEndpoints` / `listTraceCalls` and friends).
- **Indexes moved to a B-tree, killing the hot-value build cliff.** The primary
  and secondary indexes were sorted slices; they are now `tidwall/btree`. An
  index value shared by many records used to cost an O(n) memmove per insert
  (O(N²) to build) — now O(log n). Indexed writes got faster as a result
  (~9.0 µs carrying two index values). Honest trade: the grouped
  newest-per-value walk the History list uses went from ~0.1 to ~0.2 ms — still
  ~8× SQLite's `GROUP BY` and ~90× an lmdb scan.
- **Periodic hint snapshots are now O(1).** With `hintEveryN` set, every Nth
  write used to copy the whole index to snapshot it for the background hint
  writer — O(store size) per checkpoint. The B-tree's copy-on-write clone makes
  that a pointer copy, so per-write cost stays flat no matter how large the
  store grows.
- **`putMany` writes a binary batch.** Batch values ride a single concatenated
  buffer with length-prefixed metadata instead of per-value base64 / JSON
  escaping — large-value imports are ~2.8–5× faster, with no regression on
  small values. The public signature is unchanged.
- **Correctness hardening, `stats()`, and a lock-free handle registry.** Added
  `stats()` (`{ records, mainBytes, blobBytes }` — in-memory counters, no I/O)
  and fixed several edge cases surfaced by a full review: self-eviction on an
  over-cap index write, a fail-stop after a failed compaction swap, crash
  recovery past a dangling blob reference, and `Reindex` purging index values
  whose kind no longer matches the declaration. The FFI handle registry moved to
  a lock-free `sync.Map` (open-once, read-many), so concurrent calls no longer
  serialize on a registry mutex.
