# Release v0.5.1

Improves ACP-backed AI sessions so long-running agent work no longer blocks
follow-up prompts, and adds shared tracing for prompt conversations.

- Removed local prompt queueing from Codex and Claude ACP adapters, so users can
  send another prompt while an earlier agent turn is still running.
- Added shared ACP conversation tracking with generated prompt message IDs,
  lifecycle status, timestamps, streamed updates, responses, and errors.
- Added `promptConversations`, `unfinishedPromptConversations`, and
  `onConversationUpdate` to both ACP adapters for prompt lifecycle tracing.
- Shared conversation bookkeeping between Codex and Claude through
  `AcpConversationManager`, keeping adapter behavior aligned across providers.
- Added startup coordination with `runPromise`, so concurrent prompt sends do
  not create duplicate ACP sessions while an adapter is starting.
- Kept the visible `AI is thinking` animation controlled by UI prompt state
  instead of conversation tracing state, avoiding provider-specific ACP pending
  behavior leaking into the chat UI.
- Changed AI adapter resolution so `@ai` shows a clear configuration error when
  no `-ai` option or `.r1qconfig.json` `ai` value is set.
- Updated README guidance for declaring an AI adapter explicitly.
- Added `start:codex` and `start:claude` npm scripts for local provider-specific
  runs.
- Tightened AI chat layout so the input line sits closer to the bottom of the
  chat panel.
- Added ACP runtime documentation with architecture diagrams for adapter
  communication, permission handling, prompt writing, response streaming, and
  conversation management.
- Added and updated unit coverage for concurrent ACP prompts, conversation
  lifecycle tracking, undeclared AI adapter behavior, and conversation-manager
  error handling.
