# Release v0.7.0

Adds non-interactive request execution and Unix socket notifications, so
external `r1q -p` runs can update an already open terminal app.

- Added `-p` CLI support for running a single `.nts` request file without
  starting the terminal UI. Paths may include or omit the `.nts` extension.
- Added `r1q`/`ntee-r1quest -p` response output using the existing formatted
  Result-pane response renderer.
- Added Unix socket listener support through `.r1qconfig.json` `sock`, enabled
  only when a socket path is configured.
- Added external request event parsing for JSON payloads with `ntsPath`,
  `ntsFile`, `time`, and `responseContent` fields.
- Updated `-p` execution to measure request time and post the formatted response
  to the configured socket when an open terminal app is listening.
- Updated the terminal app so external request events switch the background to
  `@query`, highlight the matching request in the collection, show the external
  response in Result, and keep the AI overlay visible when `@ai` is active.
- Added Result-pane error reporting for invalid external event JSON and socket
  listener startup/runtime errors.
- Added request-root config loading, so `<request-root>/.r1qconfig.json` can
  define `sock` for runs started with `-r`.
- Added cached runtime config loading so `root`, `ai`, `sock`, and parsed CLI
  arguments are resolved once at startup and shared by runtime callers.
- Updated README documentation for `-p`, request-root config lookup, and socket
  event forwarding.
