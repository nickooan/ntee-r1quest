# Release v0.19.2

A housekeeping release: the embedded storage engine moves out to its own
project. No user-facing behavior changes — same engine, same API, now consumed
as a regular dependency.

- **ntee-db graduates to its own repository.** The Go core, C API, and Node.js
  binding (~9.7k lines) have been extracted from this repo to
  [github.com/nickooan/ntee-db](https://github.com/nickooan/ntee-db) and are now
  published on npm as [`ntee-db`](https://www.npmjs.com/package/ntee-db) (fresh
  versioning; this release consumes `^0.1.0`). The runtime's import changes from
  the workspace-local `@ntee/ntee-db` to the registry package `ntee-db`. The
  package ships the same prebuilt native libraries (darwin-arm64, linux-amd64,
  linux-arm64), so nothing new is required at install time.
- **Leaner repo and build.** With the engine gone, the `build:db` /
  `build:db-linux` steps are dropped — `npm run build` is now just
  `build:ts && build:tui` — and the npm workspace + `bundleDependencies`
  wiring for the embedded binding is removed; the Go workspace now covers only
  the TUI. Engine development, benchmarks, and its release notes continue in
  the new repository.
