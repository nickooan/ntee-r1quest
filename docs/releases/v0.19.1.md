# Release v0.19.1

A small follow-up to v0.19.0: fixes horizontal scrolling in History mode and
makes record-by-index searches cheaper by collapsing them into a single native
call.

- **History mode scrolls horizontally.** Left / Right now scroll the selected
  record's pane sideways, so a wide line (a long URL, a single-line JSON body)
  is readable past the right edge. Before, only Up / Down (vertical) and
  Shift+Up / Down (select) were bound — the arrows did nothing. Selecting a
  different record resets the horizontal scroll, and it clamps to the widest
  line. Status hint: `↑/↓/←/→ scroll · shift+↑/↓ select · s search · esc back`.
- **Record searches now use one FFI crossing.** `secIndexRecords`,
  `secIndexPrefixRecords`, and `prefixScanRecords` used to make two async native
  calls — an index query for the keys, then a separate `getMany` for the values.
  They now do the index walk and the batched read in a single native call
  (`nteedb_*_records_json`), removing one worker-thread hop and the keys
  round-trip: ~22% faster per search in an interleaved A/B (≈83 vs ≈107 µs for a
  ~20-record search), which is the shape of the History list's fetch. Same method
  signatures — a drop-in win for the app; the embedded binding moves to
  `@ntee/ntee-db` 0.4.1.
