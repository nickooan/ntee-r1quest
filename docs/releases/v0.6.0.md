# Release v0.6.0

Adds faster mode switching and editor completions for request files, with
smarter suggestions for macros, referenced definition keys, and `.ntd` refs.

- Added Shift+Tab quick mode switching through `@query`, `@view`, `@search`,
  and `@ai`, starting from the current mode and hiding the AI overlay when
  cycling back out.
- Fixed stale Result-pane file content after switching from `@view` back to
  `@query` and executing a request.
- Kept request execution idempotent from the sidebar highlight, so pressing
  Enter repeatedly reruns the selected request instead of submitting an empty
  command.
- Added shared generic key-action helpers for common input cursor movement,
  insertion, deletion, ignored modifier keys, and quick-switch detection.
- Added editor suggestions in `@edit` for request keywords, macros, referenced
  `.ntd` keys, and concrete `@i(key)` intermediate macro completions.
- Added dynamic `ref` path suggestions for `.ntd` files and matching
  directories, with per-keystroke aborts, a one-second lookup timeout, and a
  capped result set.
- Updated suggestion controls so Up/Down select suggestions while the dropdown
  is visible, and Tab or Enter applies the highlighted suggestion by replacing
  the already typed fragment.
- Updated suggestion dropdown rendering with compact reverse-color rows, a dark
  green selected row, capped visible height, and automatic scrolling to the
  selected suggestion.
- Cached referenced `.ntd` key parsing by file mtime and refreshed suggestions
  after saving because edited `ref` lines can change available macro keys.
- Added example `.ntd` data files with separate `host`, `path`, and
  `content-type` values for exercising ref suggestions.
- Split terminal behavior out of `terminal-app.tsx` into focused edit
  suggestion, AI controller, and file-navigation helpers.
- Added and updated unit coverage for quick switching, idempotent request
  execution, editor suggestions, dynamic ref suggestions, dropdown key handling,
  and terminal helper behavior.
