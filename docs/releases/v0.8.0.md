# Release v0.8.0

Improves edit-mode speed and request authoring assistance, and expands the AI
plugin into a broader R1Quest project helper.

- Added modern HTTP header suggestions in `@edit`, including common request,
  cache, auth, forwarding, idempotency, and tracing headers.
- Added context-aware `header <prefix>` suggestions that insert the selected
  header name with the trailing `, ` separator.
- Split static editor suggestion items into a dedicated module while keeping the
  existing editor-suggestion entrypoint exports compatible.
- Added `Ctrl+E` in `@view` to jump directly into `@edit` for the currently
  reviewed file, including raw terminal `Ctrl+E` input handling.
- Added `Ctrl+S` in `@edit` to save immediately and return to `@view` without
  opening the save confirmation prompt, including raw terminal `Ctrl+S` input
  handling.
- Added `Ctrl+A` in `@edit` to move the current whitespace-delimited token into
  the input bar for inline editing, without crossing line boundaries.
- Kept token selection a no-op when the edit cursor is surrounded by spaces.
- Renamed the bundled Claude Code plugin to `r1quest-ai-plugin`.
- Expanded the plugin with `r1quest-language-runtime` for `.ntd`/`.nts` syntax,
  macros, content types, and one-shot `-p` execution guidance.
- Expanded the plugin with `r1quest-project-editor` for scanning request roots
  and safely updating existing `.ntd` and `.nts` files.
- Updated the OpenAPI generator skill to respect reusable Swagger/OpenAPI
  definitions and place duplicated shared parameters in `.ntd` data files.
- Updated README documentation for the new key bindings, header suggestions, and
  renamed R1Quest AI plugin install paths.
- Removed unused `react-devtools-core` from runtime dependencies.
- Added and updated unit coverage for header suggestions, edit/view shortcuts,
  token selection, direct save, and raw terminal control-key input forms.
