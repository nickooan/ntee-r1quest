# Release v0.13.0

Adds request tracing with trace-grouped history, run-time environment overrides
and `or` macro defaults for the request language, and a smarter, adapter-aware
AI overlay, alongside edit-mode and syntax-highlighting refinements.

- Added a `-ti <id>` flag to tag a one-shot run (`-p`) with a trace id, and a new
  `traceIndex` in the cache that records every tagged call under its id in call
  order; unlike the per-endpoint history, the trace keeps each call (including
  repeats of the same endpoint) and appends atomically so concurrent runs can't
  lose an entry.
- Filtered History to a single trace with `@h <traceId>` (or
  `@history <traceId>`): the left pane lists only that trace's calls, numbered in
  the order they ran, while bare `@h`/`@history` still shows all endpoints.
- Showed the trace id in the rendered output — a `Trace:` line under the status
  in the response view and the History results — and carried it on external
  request events when present.
- Added a `-env '{"KEY":"value"}'` flag that supplies `@env(...)` values as a JSON
  object at run time, merged over `process.env` with the supplied values winning
  on duplicate keys and coerced to strings, so required variables can be passed
  without exporting them in the shell.
- Added `or` defaults to the `@i` and `@env` macros — `@i(accept or "json")`,
  `@env(PORT or 8080)`, `@env(DEBUG or false)` — used when the key or variable is
  missing. Defaults must be an immediate string, number, or boolean and apply in
  value position (body, header, auth, `.ntd` values), not inside quoted strings.
- Reworked the "thinking" indicator in AI mode to follow real activity —
  streaming output, in-progress tool calls, or a recent update — instead of the
  raw prompt turn, so it clears once the agent has replied even when a background
  job keeps the turn open, and no longer appears on a freshly opened, empty chat.
- Labelled the AI overlay with the chosen agent, so a Claude session reads
  "Claude Chat", "Claude Response", and "Claude is thinking" (and likewise for
  Codex and Cursor) instead of the generic "AI".
- Queued concurrent agent permission requests and surfaced them one at a time
  instead of overwriting a single slot, so a second request no longer orphans the
  first and stalls the prompt turn.
- Added an opt-in ACP diagnostic log enabled with `R1QUEST_ACP_DEBUG` (to
  `~/.ntee-r1quest/acp-debug.log` or a given path) that records the prompt
  lifecycle, every session update, and permission events for debugging AI
  sessions; it is off by default and best-effort.
- Changed edit-mode navigation so an accidental arrow press can't move the file
  cursor while text is being typed: with uncommitted input, Shift+arrows move the
  file cursor while plain Left/Right move within the buffer and plain Up/Down are
  ignored; with nothing buffered, plain arrows navigate the file as before.
- Highlighted the macro `or` default keyword and its immediate default value, and
  `@i(key)` macros embedded inside quoted strings, matching only the forms the
  compiler actually interpolates.
- Taught the AI one-shot-runner skill to run an ordered task of requests —
  planning the steps, sharing a trace id (`-ti`), and chaining a value from one
  response into the next via `-env` — and clarified across the skills that
  `@env(...)` belongs only in `.ntd` files.
- Added test coverage for the trace index, environment overrides, macro defaults
  (including the string-position limit), the activity-based thinking indicator
  and agent-named overlay labels, the queued permission handling, and the
  edit-mode arrow behaviour.
