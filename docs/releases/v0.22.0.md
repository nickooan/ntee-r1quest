# Release v0.22.0

A history-mode release. The history list learns to be found and to be taken
away: Ctrl+P opens a fuzzy finder over every cached endpoint — same feel as
ntee-editor's goto-anything — a single `c` copies the record you're looking
at, and the list itself now arrives newest call first instead of
alphabetically.

- **Ctrl+P fuzzy finder in history mode.** A centered overlay fuzzy-searches
  the history list by endpoint label _and_ full request URL, so a path
  fragment, a host name, or a query-string scrap all narrow the list. Matched
  characters light up bold orange, a live `n/m` count tracks the filter, ↑/↓
  walk the matches, Enter selects — the sidebar highlight jumps to the entry
  and the right pane opens its request and response at the top — and Esc
  backs out without moving the selection. An empty query lists everything, so
  Ctrl+P doubles as a walkable picker. The matcher is ntee-editor's
  goto-anything scorer ported into a new `tui/internal/fuzzy` package:
  case-insensitive subsequence matching with word-boundary and consecutive-run
  bonuses, an allocation-free pre-filter, and match-position highlighting
  computed only for visible rows — it filters a 50k-entry corpus in ~25ms, so
  every keystroke over a real history list is effectively instant, entirely
  inside the Go TUI.
- **History arrives newest first.** The runtime's endpoint list is now ordered
  by most recent call, most recent on top, instead of alphabetically by label
  — the endpoint you just hit is the first row. Under the hood it asks the
  endpoint index for exactly the latest record per endpoint (the grouped `-1`
  query) rather than scanning and deduping every cached call record.
- **`c` copies the current record.** In history mode, `c` puts the right
  pane's formatted request and response — URL, method, headers, both bodies —
  on the system clipboard, with the same "copied" notice as `@copy`. The
  notice clears on the next keystroke, and the status line now lists
  `ctrl+p find` and `c copy` alongside the existing keys.
- **`@cp` is an alias for `@copy`.** Works everywhere `@copy` does, including
  when typed from AI mode.
