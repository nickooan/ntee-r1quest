# Release v0.20.1

A UX-focused follow-up to the joint-chain release. The editor learns to jump
between files: put the cursor on a `ref`'d `.ntd`, an `@run` target, an `@f`
upload, or an `@i(key)` and press Ctrl+J to open the file (or the exact line
that defines the key), then Ctrl+O to jump back. The AI chat stops being a wall
of one-color text — user turns, agent prose, tool activity, and background-task
notifications now render distinctly, agent replies pick up light markdown
styling for both Claude and Codex, and the status bar tells you which keys and
mode-switches are available.

- **Jump to file references in the editor (Ctrl+J / Ctrl+O).** In edit mode the
  token under the cursor — or under a Ctrl+A line selection — is classified and
  followed: a `ref ../a.ntd` path, an `@run(<path>)` step target, an `@f(<path>)`
  upload, or an `@i(key)` reference. Paths open relative to the file being
  edited (extension optional for `@run`); an `@i(key)` jumps straight to the
  `.ntd` line that _defines_ that key. Key resolution matches runtime
  semantics — ref'd `.ntd` files merge in order and later files override
  earlier ones, so the jump lands on the last definition that actually wins
  (`suggest.ResolveKeyDefinition` / `FindDefinitionKeyLine`). Ctrl+O pops a
  jump-back stack (up to 20 frames) that restores the previous file, cursor,
  and scroll position. The edit status line advertises it:
  `Ctrl+S save · Ctrl+F find · Ctrl+J/O jump/back · Ctrl+Z undo · esc discard`.
- **The AI chat gets color.** Messages now render by kind instead of one flat
  color: user turns are bold, agent prose is plain, and tool / background-task
  activity renders as a `⏺` bullet whose color tracks status — yellow while
  running, green on success, red on failure — with the tool title dimmed. Tool
  calls are tracked by id and updated _in place_ as their status changes, so a
  long-running step is one line that turns from yellow to green rather than a
  pile of repeated `[Title]` notices.
- **Agent replies read like markdown — for Claude and Codex both.** Assistant
  prose picks up light accents: `#` headers, `` `code` ``, fenced code blocks,
  `**bold**`, `*italic*`, `***bold italic***`, `~~strikethrough~~`, `-`/`*`/`+`
  bullets, numbered lists, blockquotes, `[label](url)` links (label styled, the
  URL kept as a dim, copyable parenthetical), and bare URLs (with trailing
  sentence punctuation left outside the link). Underscore emphasis (`_x_`,
  `__x__`) is deliberately _not_ styled so snake_case and dunder identifiers
  (`foo_bar`, `__init__`) in agent prose never get mangled. Styling is
  streaming-safe — a span only lights up once its closing delimiter arrives on
  the same line — and wrapping happens on the underlying text so accents
  survive line breaks without leaking escape codes. Both agents flow through
  the same message path, so the same rendering applies to each.
- **Background-task notifications render as text, not XML.** Claude Code injects
  background-task lifecycle events as synthetic user messages carrying a raw
  `<task-notification>…</task-notification>` blob. These are now parsed into a
  friendly `⏺` line — the human summary plus the `output:` file path, colored by
  status (a killed/failed task shows red) — instead of dumping the markup into
  the transcript. Because resumed sessions replay through the same path, the
  cleaned-up rendering shows for both live runs and reopened history.
- **The status bar says what you can do.** AI mode gains a faint hint row under
  the input: `Ctrl+J newline · enter send · ↑/↓ scroll · esc back · shift+tab →
query`. Every primary mode also advertises where Shift+Tab goes next
  (`query → history → ai → query`) on its own compact row — so the mode cycle is
  discoverable without crowding what you're typing.
