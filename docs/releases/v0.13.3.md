# Release v0.13.3

Adds a Result-pane clipboard command and a Claude Code plugin installer,
restyles the live response view to match the history view, records failed
responses in history, and adds an install-from-source script — alongside a
refreshed demo and documentation.

- Added `@report` (alias `@copy`) to copy the Result pane content to the system
  clipboard on macOS and Linux. It copies exactly what the pane shows — the
  response, an open file, or a history record — excluding the pending animation,
  and confirms with a "Copied" overlay (or a "Copy failed" overlay when there is
  nothing to copy or no clipboard tool is available); Enter dismisses either.
- Restyled the live response view to match the history view: a `<path> [METHOD]`
  summary, `── Request ──` / `── Response ──` section rules, aligned `URL` /
  `Method` / `Status` fields, and indented headers and body. Shared layout
  primitives now back both views.
- Recorded failed responses in history: a request that returns a non-2xx status
  (4xx/5xx) is now saved to the per-endpoint history and the trace index, not
  only 2xx responses. Pure runtime errors that never produced a response (network
  failures, bad config) are still skipped. Applies to both the app and one-shot
  `-p` runs.
- Added an `install.sh` to install and update directly from the Codeberg source
  and link it into the global npm — for tracking `main` or a pinned tag. Re-run
  it to update; overridable via `NTEE_REF` / `NTEE_REPO` / `NTEE_DIR`, with a
  `~/.local/bin` fallback when the global npm prefix is not writable.
- Added `--install-claude-plugin`, which installs the bundled R1Quest AI plugin
  into Claude Code through the `claude` CLI (registers the marketplace, then
  installs the plugin). npm-registry installs do not carry the plugin source and
  are pointed to the Codeberg source instead.
- Added npm `keywords` for discoverability, refreshed the README (AI
  collaboration in `@ai` mode, install-from-source, macOS/Linux-only support, the
  CLI reference), and regenerated the demo GIF to match the current UI — the new
  response format and the agent-named AI overlay — with the version read from
  `package.json`.
- Added test coverage for the clipboard platform selection, the restyled response
  output, failed-response recording, the Claude plugin installer, and the
  `@report` command.
