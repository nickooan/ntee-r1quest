# Release v0.5.0

Improves file review/edit workflows in the main terminal layout and adds
keyboard-first navigation for request collections.

- Replaced the view/edit overlay with inline Result-pane file review and edit
  rendering, so selected files now display in the main terminal content area.
- Updated pane titles by mode: `@query` shows `Result`, `@view` shows
  `Reviewing`, `@edit` shows `Editing`, and `@search` shows `Searching`.
- Kept `@search` working against viewed files, including jumping to the
  matching line before entering `@edit`.
- Added cursor editing to `@query`, `@view`, and `@search` inputs, including
  Shift+Left/Right cursor movement, insertion at the cursor, and backspace
  before the cursor.
- Added Shift+Up/Down sidebar selection in `@query` and `@view`, so users can
  move through the current collection directory without typing each path.
- Changed directory selection so Shift+Up/Down only moves the highlight; a
  directory expands only after pressing Enter.
- Added Esc parent-directory navigation in `@query` and `@view`, so paths like
  `request/folder-1/` fold back to `request/`.
- Preserved active edit-file Esc behavior for the save prompt.
- Improved sidebar highlighting so partial child input under an expanded
  directory keeps the closest parent directory highlighted until a child match
  exists.
- Renamed base-mode key helper code to query-mode naming to match the actual
  `@query` mode behavior.
- Added and updated unit coverage for inline file layout, sidebar selection,
  parent-directory navigation, cursor editing, and highlight fallback behavior.
