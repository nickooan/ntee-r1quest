# Release v0.14.0

Adds resumable AI sessions with a session picker, marks replayed history with a
divider, restyles the AI input bar, refactors the cache into focused modules,
hardens the install script against private registries, and teaches the OpenAPI
generator to always emit a `content-type` header.

- Added resumable AI sessions. The first `@ai` of a run now shows a session
  picker when prior sessions exist — a keyboard-navigable list with "New
  session" default-selected at the top and past sessions (with their last-used
  time) below; Enter resumes the highlighted one and Esc cancels. Resuming
  replays the conversation through the agent's `loadSession` capability across
  the Claude Code, Codex, and Cursor adapters, falling back to a fresh session
  when an agent does not advertise that capability so resume never breaks
  startup.
- Persisted AI sessions in a per-agent system store (`<agent>-session`),
  stamped with `createdAt`/`updatedAt`. A session is bumped on each resume, and
  a startup cleanup prunes sessions whose last use has aged past the configured
  cleanup period so the resume list never offers dead sessions.
- Added the `session-cleanup-period` config key — the number of days an AI
  session is kept before startup cleanup prunes it (default `7`).
- Added a history divider on resume: replayed past turns are followed by an
  `── above is history ──` rule line so it is clear what came from the previous
  session and what is live. Replayed user turns now render as their own user
  messages, so each assistant reply stands as a separate block instead of
  merging consecutive replies together.
- Restyled the AI overlay: the input bar now uses a darker background
  (`#0a0a0a`) than the overlay so it reads as a distinct field, alongside a
  refreshed overlay border.
- Refactored the cache into focused modules — `api`, `clear`, `input`, `store`,
  `system`, and `trace` — replacing the monolithic index, and added the system
  store that backs the new session records.
- Hardened `install.sh` to force the official npm registry for the dependency
  install and build, so a local or private registry configured in the
  environment cannot break the from-source build; the override is scoped to
  those commands and the user's npm config is left untouched.
- Updated the OpenAPI generator skill to always emit a `content-type` header on
  requests that send a body, sourced from a referenced `.ntd` key, the exact
  media type read from the spec, or the `@i(content-type or "...")` default
  form, with a matching validation check.
- Added test coverage for the AI session store and replay, the divider and
  per-turn user messages, and the restyled AI input.
