# Release v0.10.0

Moves R1Quest configuration to YAML and adds interactive setup, configurable
editor suggestions, and expanded AI plugin support.

- Replaced `.r1qconfig.json` with `.r1qconfig.yaml` and `.r1qconfig.yml`
  configuration parsed through the `yaml` package.
- Changed the home config path to `~/.ntee-r1quest/r1qconfig.yaml`.
- Added `--init`, which opens an Ink config generator for selecting a collection
  path and optional Codex or Claude AI agent.
- Added `--version`, which prints the current app version and exits without
  opening the terminal app.
- Added `custom-suggestions` configuration for user-defined request header and
  body object key suggestions.
- Added context-aware custom suggestion insertion, using `header <key>, ` for
  headers and `<key>: ` for body object keys.
- Added an example YAML config and removed the obsolete JSON config template.
- Updated the bundled Claude Code plugin to use a local marketplace with the
  plugin stored under `plugin/`.
- Updated Claude Code marketplace installation and Codex skill import
  documentation.
- Added the `r1quest-one-shot-runner` skill for locating and executing one named
  request from an existing collection.
- Added coverage for YAML config loading, ignored JSON configs, interactive
  config generation, custom suggestions, and editor insertion behavior.

Example custom suggestions:

```yaml
custom-suggestions:
  - some-style-id
  - x-trace-token
```
