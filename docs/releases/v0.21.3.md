# Release v0.21.3

A maintenance release that refreshes the ACP (Agent Client Protocol) stack to
the current upstream releases. No user-facing behavior changes.

- **`@agentclientprotocol/sdk` 0.22.1 → 1.3.0.** The protocol SDK crosses its
  1.0 line; the adapters now build against the stable 1.x request/response
  types.
- **`@agentclientprotocol/claude-agent-acp` 0.36.1 → 0.60.0.** Picks up the
  latest upstream Claude Code ACP agent.
- **`@agentclientprotocol/codex-acp` 1.1.0 → 1.1.7.** Latest upstream Codex
  ACP agent.
- **Adapted to the SDK 1.x prompt API.** The 1.x protocol dropped the
  `messageId`/`userMessageId` echo on prompt requests and responses, so the
  Claude Code, Codex, and Cursor adapters no longer send or store it.
  Conversation tracking already keyed off locally generated conversation ids,
  so nothing changes in practice; the adapter tests and docs were updated to
  match.
